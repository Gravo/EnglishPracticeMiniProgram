# CEO Agent Operating System

**项目**: 高中英语泛读泛听 MVP  
**当前阶段**: MVP1，10 个高考词，每词 7 个语境卡  
**CEO 代理目标**: 调度产品、内容、技术、质量、运营、商业化子 agent，让最小闭环尽快跑到真实学生手里。

---

## 1. CEO 代理职责

CEO 代理不亲自陷入所有细节，而是负责四件事：

1. 明确本轮目标：先证明高中生愿意每天刷英语语境卡。
2. 拆分任务：让每个子 agent 只做一个领域，文件和责任边界清晰。
3. 设置控制点：词表、语料、音频、发布、付费、营销都要有人工审批闸门。
4. 看指标复盘：一周后决定扩词、改产品、做付费或暂停。

## 3. 七次曝光规则

本 MVP 采用“7 次不同语境曝光”作为产品规则，而不是声称看 7 次必然掌握。

一个词的 7 张卡：

1. `meaning`：基础含义
2. `campus`：校园/学习场景
3. `family`：家庭/日常场景
4. `news` 或 `science`：新闻/科普场景
5. `dialogue`：口语对话
6. `cloze`：主动回忆填空
7. `transfer`：迁移复习，帮助学生把词用到新语境

掌握状态不能只看“看过”。MVP 规则：

- 至少看过 7 个不同语境
- 至少完成 2 次主动回忆卡，例如 cloze 或听写
- 至少跨 3 天出现
- 最近一次复习不能答错

## 4. 人工控制点

| 控制点 | 触发时间 | 审批人 | 不通过就做什么 |
|---|---|---|---|
| 词表入选 | 每批词进入 MVP 前 | CEO + 词汇负责人 | 降级或移出本批 |
| 语料审核 | 生成卡片后 | 质量/合规 Agent + CEO 抽检 | 改写、删除、换场景 |
| 音频抽检 | TTS 生成后 | CEO 抽检 10%，学生听感反馈 | 重生成或换音色 |
| 内容包发布 | 上传服务器前 | CEO | 停止发布，修 manifest |
| 小程序发版 | 真机测试后 | CEO | 修 bug，不提交审核 |
| 试用扩量 | 10 人试用完成后 | CEO | 留存不足则改产品 |
| 付费上线 | 免费试用数据达标后 | CEO | 暂缓收费，继续验证 |
| 营销投放 | 有留存和转化信号后 | CEO | 不投放或只做免费内容 |

## 5. MVP1 决策原则

只做能验证核心假设的功能：

- 学生愿不愿意刷？
- 7 个语境是否帮助理解？
- 家长是否看得懂价值？
- 小程序是否能稳定加载远端内容？
- 内容生产是否能规模化？

暂时不做：

- 大规模影视片段上线
- 复杂 AI 推荐
- 老师班级后台
- 完整支付系统
- 自动生成 3677 词全量内容

## 6. 每周 CEO 看板

内容侧：

- 本周新增词数
- 本周新增卡片数
- 审核通过率
- 音频生成成功率
- 被打回原因 Top 5

产品侧：

- 日活
- 次日留存
- 人均完成卡片数
- 7 语境完成率
- cloze 正确率

商业侧：

- 试用人数
- 愿意付费人数
- 家长正反馈数
- 预估月成本
- 是否达到投放条件
